## Internal Linking Architecture

```
Homepage (/{lang}/)
├── Category pages (/{lang}/{category}/) -- 7 categories, pillar pages
│   └── Individual guides/reviews/comparisons
├── Hub page (/{lang}/all/) -- master index of ALL content
│   ├── Grouped by category with anchor links
│   ├── Links to category pages (heading links)
│   └── Search bar filters all content types
├── About page
└── Legal page
```

**Max click depth:** 2 clicks from homepage to any guide.
**Crawl paths:** Google can reach every page via homepage > category OR homepage > hub.

## External Linking Strategy

- Link to official product pages when mentioning products
- Link to official help/support docs when referencing procedures
- Use `rel="noopener" target="_blank"` for external links
- 2-4 external links per guide to authoritative sources
- Signals trust and topical relevance to Google
- Especially important for YMYL-adjacent content (security guides)

3. `git commit + push`

The content cron (when wired) will call `deploy.sh` after generating new articles.
