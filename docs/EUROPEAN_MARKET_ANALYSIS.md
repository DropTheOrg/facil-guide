## Part 3: IP-Based Language Routing (Technical Spec)

### Middo's Insight
"Technically, seniors should not even see the site as multilingual. They usually don't speak multiple languages. The language switch could happen in the footer only."

This is brilliant UX thinking. Here's how to build it:

### Architecture: Cloudflare Worker + Astro

**Current URL structure:** `facil.guide/{lang}/{slug}/` (e.g., `facil.guide/fr/utiliser-siri-iphone/`)

**New URL structure:** `facil.guide/{slug}/` (single canonical URL)

**How it works:**

1. Visitor hits `facil.guide/use-siri-iphone/`
2. Cloudflare Worker intercepts the request
3. Worker reads `request.cf.country` (free, automatic with Cloudflare)
4. Country maps to language:
   ```
   FR → fr
   ES, MX, AR, CO, CL, PE → es
   PT, BR → pt
   IT → it
   Everything else → en
   ```
5. Worker rewrites (NOT redirects) to `/{lang}/{slug}/`
6. Astro serves the correct language version
7. User sees `facil.guide/use-siri-iphone/` in their browser — no `/fr/` prefix visible

**The "Send Link to Dad" Flow:**

1. Middo (in Portugal) reads `facil.guide/use-siri-iphone/` → sees EN (because his browser language is EN, or we detect his preference)
2. Middo copies the URL and sends it to his French dad
3. Dad opens the SAME URL in France → Cloudflare detects `country: FR` → serves French version
4. Dad sees the guide in perfect French, never sees a language picker, never knows other versions exist
5. If Middo sends the same URL to his Spanish grandmother → she sees Spanish

**Edge cases:**
- VPN users: fallback to `Accept-Language` header
- Bilingual countries (Belgium, Switzerland, Canada): use `Accept-Language` as tiebreaker
- Manual override: tiny flag icon in footer. Clicking sets a cookie that overrides geo-detection.
- SEO: `hreflang` alternate links in `<head>` still exist for Google. `rel="canonical"` points to the language-neutral URL.

### Implementation

```javascript
// Cloudflare Pages Function: functions/[[path]].js
const COUNTRY_LANG = {
  FR: 'fr', MC: 'fr', // France, Monaco
  ES: 'es', MX: 'es', AR: 'es', CO: 'es', CL: 'es', PE: 'es', VE: 'es', EC: 'es', GT: 'es', CU: 'es', BO: 'es', DO: 'es', HN: 'es', PY: 'es', SV: 'es', NI: 'es', CR: 'es', PA: 'es', UY: 'es',
  PT: 'pt', BR: 'pt', AO: 'pt', MZ: 'pt',
  IT: 'it', SM: 'it', VA: 'it',
  // Belgium: check Accept-Language (FR or NL)
  // Switzerland: check Accept-Language (FR, DE, IT)
  // Canada: check Accept-Language (FR or EN)
};

export async function onRequest(context) {
  const { request } = context;
  const url = new URL(request.url);

  // Skip if already has language prefix
  if (/^\/(en|fr|es|pt|it)\//.test(url.pathname)) {
    return context.next();
  }

  // Check cookie override first
  const cookies = request.headers.get('Cookie') || '';
  const langCookie = cookies.match(/facil_lang=(\w{2})/);
  if (langCookie) {
    url.pathname = `/${langCookie[1]}${url.pathname}`;
    return context.next({ request: new Request(url, request) });
  }

  // Geo detection
  const country = request.cf?.country;
  let lang = COUNTRY_LANG[country] || 'en';

  // For multilingual countries, check Accept-Language
  if (['BE', 'CH', 'CA', 'LU'].includes(country)) {
    const acceptLang = request.headers.get('Accept-Language') || '';
    if (acceptLang.includes('fr')) lang = 'fr';
    else if (acceptLang.includes('it')) lang = 'it';
    else if (acceptLang.includes('es')) lang = 'es';
    else if (acceptLang.includes('pt')) lang = 'pt';
    else lang = 'en';
  }

  // Rewrite (not redirect) to language version
  url.pathname = `/${lang}${url.pathname}`;
  return context.next({ request: new Request(url, request) });
}
```

### What The Senior Sees

- URL: `facil.guide/use-siri-iphone/` (clean, simple, no language codes)
- Content: fully in their language
- No language picker visible (it's in the footer, small, for the helper who might need it)
- No "choose your country" popup
- No cookie consent wall (we don't track anything)

### What Google Sees

- `hreflang` alternates for all 5 versions in `<head>`
- Each language version accessible at `/{lang}/{slug}/` for crawlers
- Sitemap includes all language variants
- `rel="canonical"` on the geo-served version points to the language-specific URL

4. **No country-specific content.** WikiHow has nothing for France Connect, SPID, Chave Movel Digital. They can't write these because they don't have local expertise.

5. **Ad-heavy UX.** Popups, newsletter begging, interstitials. For a senior with trembling hands, one misclick on an ad and they're lost.

## Part 5: Community & User Base Model

### The Facil Community (Inspired by Middo's Vision)

**Core idea:** Facil isn't just a library — it becomes a platform where helpers and tech-savvy volunteers contribute, get recognized, and build a community around helping seniors.

### User Roles

1. **Reader** (no account needed) — 95% of visitors. Just reads guides. No friction.
2. **Helper** (free account) — saves guides, sends to parents, tracks what they've sent. Gets a "helper dashboard."
3. **Contributor** (earned role) — submits guide suggestions, translations, corrections. Earns points.
4. **Expert** (invited/earned) — writes full guides, reviews community submissions. Gets profile page.

### Point System: "Facil Points"

| Action | Points | Notes |
|--------|--------|-------|
| Sign up | 10 | Welcome bonus |
| Send a guide to someone | 5 | "Send to parent" button |
| Report an error | 10 | Corrections keep quality high |
| Suggest a guide topic | 15 | Community-driven content pipeline |
| Submit a translation correction | 25 | Native speakers improve translations |
| Write a guide draft | 100 | Reviewed by Expert before publishing |
| Guide approved and published | 250 | Attribution on the guide page |
| Refer another helper | 50 | Viral growth |

### Leaderboard & Recognition

- Monthly leaderboard: "Top Helpers This Month"
- Badges: "France Helper" (helped 10 people in France), "Multilingual" (contributed in 2+ languages), "Guardian" (5 scam guides sent)
- Profile page showing: guides contributed, points earned, people helped, languages
- Annual "Facil Awards" — top contributor per country per year

### Why This Works

1. **Helpers WANT recognition.** Being tech support for your parents is thankless. A point system and badges give them social proof: "I've helped 50 people."
2. **Contributors improve quality for free.** Native speakers fix our translations. Local experts add country-specific details. We don't pay, they earn status.
3. **Content pipeline feeds itself.** Community suggests topics → we see demand → we write guides → community distributes them.
4. **Network effects.** Each helper brings their parent. Each parent tells their friends at the senior center. Each friend's child discovers Facil.

### What We DON'T Do (Anti-Patterns)

- No gamification dark patterns (no streak guilt, no pushy notifications)
- No paywall on ANY content (points buy recognition, not access)
- No user-generated guides without expert review (quality control)
- No social feed (this is not Facebook, it's a utility)
- No forum (forums die without moderation; we don't have the team)
